1️⃣ Branch ↔ Semester
Relationship Type:

One-to-Many

Meaning:

One Branch has many Semesters

One Semester belongs to one Branch

Real Example:

CSE → Semester 1, 2, 3, 4, 5, 6, 7, 8
ECE → Semester 1, 2, 3...
In Database:
semester table has a branch_id foreign key.

2️⃣ Semester ↔ Subject
Relationship Type:

One-to-Many

Meaning:

One Semester has many Subjects

One Subject belongs to one Semester

Example:

Semester 3 (CSE) →
Data Structures
Digital Logic
Discrete Mathematics

In Database:
subject table has semester_id foreign key.


3️⃣ Subject ↔ Resource
Relationship Type:

One-to-Many

Meaning:

One Subject can have many resources

One Resource belongs to one subject

Example:

Data Structures →
Unit 1 Notes
Unit 2 Notes
2021 PYQ
2022 PYQ

In Database:
resource table has subject_id foreign key.


4️⃣ User ↔ Resource
Relationship Type:

One-to-Many

Meaning:

One User (Admin) can upload many resources

One Resource is uploaded by one user

Example:
Admin1 uploads:
10 Notes
5 PYQs

In Database:
resource table has uploaded_by foreign key.


#The project follows a hierarchical academic structure. A Branch contains multiple Semesters, each Semester contains multiple Subjects, and each Subject contains multiple Resources like Notes and PYQs. Additionally, Resources are uploaded by Admin users, creating a one-to-many relationship between User and Resource. All relationships are implemented using One-to-Many and Many-to-One associations in JPA.
